`@Envapt` binds a class property to an environment variable. It's the class-based counterpart to the [`Envapter`](/docs/envapter) readers: the same parsing, converters, and cache, declared as typed fields instead of method calls.

```ts twoslash
import { Envapt, Converters } from 'envapt';
// ---cut---
class Config {
    @Envapt('PORT', { converter: Converters.Number, fallback: 3000 })
    static readonly port: number;

    @Envapt('DEBUG', { converter: Converters.Boolean, fallback: false })
    static readonly debug: boolean;
}

Config.port;
//     ^?
```

The value is read and converted once, on first access, then cached.

## Declaring decorated fields

The right form depends on whether the field is static or instance.

- **Static** fields use a plain `static readonly x: T`, with no `declare` and no initializer.
- **Instance** fields use `declare readonly x: T`, with no initializer.

`@Envapt` installs a getter with `Object.defineProperty`, and two compiler behaviors decide where that getter has to sit.

For an instance field, `useDefineForClassFields` (on by default for modern targets) emits a per-instance assignment in the constructor that would overwrite the getter with `undefined`. `declare` makes the field type-only, so nothing is emitted and the getter survives.

For a static field the reverse holds. A `declare static` member is erased, so tsc emits the decorator against the prototype and a static read never reaches the getter, returning `undefined`. A plain `static readonly` emits a real static member, so the decorator lands on the constructor where the static read finds it.

<Callout type="warn">
    Match the form to the field. Use `static readonly x: T` for static and `declare readonly x: T` for instance. A
    `declare static` field reads back as `undefined` under tsc (esbuild, swc, oxc, bun, and deno are unaffected). A plain
    instance field, or any initializer, also reads back as `undefined` because the emitted field clobbers the getter.
</Callout>

You also need `experimentalDecorators` enabled.

```json title="tsconfig.json"
{
    "compilerOptions": {
        "experimentalDecorators": true
    }
}
```

Runtime-specific decorator notes (Bun running a `.ts` entry directly, Deno's flag) live on [Compatibility](/docs/compatibility).

## Converters

The `converter` option takes the same values as [`getUsing`/`getWith`](/docs/converters): a built-in token, a primitive constructor, a custom function, or `Converters.array(...)`. A `fallback` removes `undefined` from the value and must match the converter's type.

```ts twoslash
import { Envapt, Converters } from 'envapt';
// ---cut---
class Config {
    // duration token: a time-string fallback is coerced to milliseconds
    @Envapt('CACHE_TTL', { converter: Converters.Time, fallback: '15m' })
    static readonly cacheTtl: number;

    // array token
    @Envapt('ALLOWED_ORIGINS', { converter: Converters.array({ of: Converters.String }), fallback: [] })
    static readonly origins: string[];

    // custom function: returns a shape the array builder can't, here a deduped Set
    @Envapt('TRUSTED_IPS', { converter: (raw) => new Set((raw ?? '').split(',').filter(Boolean)) })
    static readonly trustedIps: Set<string>;
}
```

With no `converter` and no `fallback`, the property resolves to the raw string or `null`.

## Shorthand decorators

`@EnvNum`, `@EnvStr`, `@EnvBool`, `@EnvUrl`, and `@EnvTime` are shorthands for `@Envapt` with a fixed converter. The call site is the key and an optional fallback.

```ts twoslash
import { EnvNum, EnvStr, EnvBool, EnvUrl, EnvTime } from 'envapt';
// ---cut---
class Config {
    @EnvNum('PORT', 3000)
    static readonly port: number;

    @EnvStr('AWS_REGION', 'us-east-1')
    static readonly region: string;

    @EnvBool('DEBUG', false)
    static readonly debug: boolean;

    @EnvUrl('APP_URL', new URL('http://localhost:3000'))
    static readonly url: URL;

    @EnvTime('CACHE_TTL', '15m')
    static readonly cacheTtl: number;
}
```

The second argument is the fallback, typed to the converter: `@EnvUrl` takes a `URL`, `@EnvTime` takes a millisecond number or a time string, the rest take their primitive. Omit it and the property resolves to the converted value or `null`. The key can also be an [ordered list](#ordered-keys).

These install the same getter as `@Envapt`, so the same [declaration rule](#declaring-decorated-fields) applies. They take a fallback only. For `required`, a `schema`, an array, or a custom function, use `@Envapt` with the options object.

## Required values

Pass `required: true` to throw `EnvaptError` on a missing or empty value instead of returning a fallback. It's mutually exclusive with `fallback`.

```ts twoslash
import { Envapt, Converters } from 'envapt';
// ---cut---
class Config {
    @Envapt('DATABASE_URL', { converter: Converters.Url, required: true })
    static readonly databaseUrl: URL;
}
```

The throw happens on first access to the property, independent of global strict mode. See [Strict mode](/docs/strict-mode) for the global switch and how `required` relates to it.

<Callout type="warn">
    First access can be deep in a request, not at boot. A missing required value then surfaces as a runtime error
    mid-request. Read the property once during startup to fail early.
</Callout>

## Validation with a schema

Pass a `schema` to validate through zod, valibot, arktype, or a hand-rolled `StandardSchemaV1`. The property type is the schema's output.

```ts twoslash
import { z } from 'zod/v4';
import { Envapt } from 'envapt';
// ---cut---
class Config {
    @Envapt('STRIPE_SECRET_KEY', { schema: z.string().startsWith('sk_') })
    static readonly stripeKey: string;
}
```

`schema` and `converter` are mutually exclusive. See [Standard Schema](/docs/standard-schema) for the full behavior.

## Ordered keys

Like the functional readers, the key can be an ordered list. The first key with a value wins.

```ts twoslash
import { Envapt, Converters } from 'envapt';
// ---cut---
class Config {
    // CANARY_URL if set, otherwise APP_URL
    @Envapt(['CANARY_URL', 'APP_URL'], { converter: Converters.Url })
    static readonly url: URL | null;
}
```

## Static and instance properties

`@Envapt` works on both static and instance properties, and the class does not need to extend anything. The cache is keyed per `ClassName.property`, so a static and an instance property of the same name never collide.

Extend `Envapter` when you also want the reader methods on the same class:

```ts twoslash
import { Envapt, Converters, Envapter } from 'envapt';
// ---cut---
class Config extends Envapter {
    @Envapt('PORT', { converter: Converters.Number, fallback: 3000 })
    declare readonly port: number;

    // mix decorator fields with reader calls
    readonly region = this.get('AWS_REGION', 'us-east-1');
}

const config = new Config();
config.port;
//     ^?
```

A misconfigured options object throws `EnvaptError` as the decorator is applied, before any property access: `required` together with a `fallback`, or a non-`StandardSchemaV1` value passed as `schema`. See [Errors](/docs/errors) for the codes.
